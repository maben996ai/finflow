# FinFlow — 金融信息聚合平台 产品需求文档（PRD）

> **版本：** v1.0  
> **日期：** 2026-04-11  
> **状态：** 草案

---

## 一、产品概述

### 1.1 背景

港美股投资者日常需要关注大量财经博主的视频内容，这些内容分散在 B 站、YouTube 等多个平台上。手动逐个平台刷新检查效率低下，容易遗漏重要信息。同时，金融资讯、券商交易数据等信息也散落在不同渠道，缺乏统一的获取入口。

### 1.2 产品定义

FinFlow 是一个金融信息聚合平台（Web 端），帮助投资者在一个页面上集中获取关注博主的最新视频动态，并通过飞书机器人实时推送通知。后续将逐步扩展为覆盖金融资讯、券商数据、AI 智能分析的一站式投资信息平台。

### 1.3 目标用户

- 港美股个人投资者，初期使用人数约 5-10 人（本人及朋友）
- 关注 5-20 位 B 站 / YouTube 财经博主
- 已建立飞书群和飞书机器人，习惯通过飞书接收消息

---

## 二、迭代规划

### 一期：信源获取 + Web 展示 + 飞书通知（本次交付）

聚焦核心链路：抓取博主视频 → Web 页面展示 → 飞书推送通知。

### 二期：金融资讯 + 多渠道扩展

接入金融资讯网站 API 和券商交易信息，丰富信息源类型。扩展通知渠道（微信、邮件等）。支持更多视频平台。Web 端支持嵌入式视频播放。

### 三期：AI 智能分析

接入 LLM 对信息源进行分类、摘要、情绪分析。自动生成每日投资信息简报。基于用户偏好的智能推荐。

---

## 三、一期功能范围

### 3.1 博主管理

**做什么：** 用户在 Web 页面上管理关注的博主列表。

| 功能点 | 说明 | 优先级 |
|--------|------|--------|
| 添加博主 | 粘贴博主主页链接（B 站 / YouTube），系统自动识别平台和博主信息（名称、头像） | P0 |
| 博主列表 | 展示所有已关注博主，显示平台、名称、头像、最近更新时间 | P0 |
| 删除博主 | 取消关注某个博主 | P0 |
| 编辑备注 | 为博主设置自定义备注名 | P1 |
| 通知开关 | 按博主单独开启/关闭飞书通知推送 | P1 |

**不做什么（一期）：** 不做博主分组/标签管理，不做批量导入。

---

### 3.2 视频源抓取

**做什么：** 定时检测关注博主是否发布新视频，将新视频信息入库。

| 功能点 | 说明 | 优先级 |
|--------|------|--------|
| B 站视频抓取 | 获取指定 UP 主最新发布的视频列表 | P0 |
| YouTube 视频抓取 | 获取指定频道最新发布的视频列表 | P0 |
| 定时检查 | 每 30 分钟执行一次全量检查 | P0 |
| 新视频识别 | 对比已入库数据，识别并标记新发布的视频 | P0 |
| 抓取容错 | 单个博主抓取失败不影响其他博主，失败自动重试（最多 3 次） | P0 |
| 抓取降级 | 当 API 不可用时，降级为推送博主主页链接 | P1 |

**抓取内容：** 视频标题、封面图 URL、视频链接、发布时间、博主名称、来源平台。

**不做什么（一期）：** 不抓取视频内容/字幕，不做视频下载。

---

### 3.3 Web 信息流

**做什么：** 以时间线形式展示所有关注博主的最新视频。

| 功能点 | 说明 | 优先级 |
|--------|------|--------|
| 视频卡片 | 展示封面图、标题、博主名称+头像、发布时间、平台标识 | P0 |
| 时间线排序 | 按发布时间倒序排列 | P0 |
| 点击跳转 | 点击卡片跳转到原平台观看视频 | P0 |
| 平台筛选 | 按平台筛选（全部 / B 站 / YouTube） | P0 |
| 已读标记 | 手动标记视频为已读，已读视频视觉上区分显示 | P1 |
| 收藏功能 | 收藏视频，可在收藏列表中查看 | P1 |
| 分页加载 | 滚动到底部自动加载更多历史视频 | P1 |

**不做什么（一期）：** 不做嵌入式视频播放，不做搜索功能，不做博主维度的聚合视图。

---

### 3.4 飞书通知

**做什么：** 博主发布新视频时，通过飞书机器人即时推送通知到群聊。

| 功能点 | 说明 | 优先级 |
|--------|------|--------|
| 新视频推送 | 检测到新视频后立即推送到飞书群，每条视频单独一条消息 | P0 |
| 消息内容 | 包含：博主名称、平台标识、视频标题、封面图、视频链接 | P0 |
| Webhook 配置 | 在 Web 设置页配置飞书机器人 Webhook URL | P0 |
| 去重推送 | 同一视频不重复推送 | P0 |
| 免打扰时段 | 可设置夜间免打扰时段，期间不推送（积压到时段结束后统一推送） | P2 |

**不做什么（一期）：** 不做微信/邮件/Telegram 等其他通知渠道，不做合并推送模式。

---

### 3.5 用户认证

**做什么：** 支持多用户登录，每位用户拥有独立的关注列表和设置。

| 功能点 | 说明 | 优先级 |
|--------|------|--------|
| 注册 | 邮箱 + 密码注册 | P0 |
| 登录 | 邮箱 + 密码登录 | P0 |
| 保持登录 | Token 认证，合理有效期内免重复登录 | P0 |
| 独立数据 | 每位用户的关注列表、已读/收藏状态、通知配置互相独立 | P0 |

**不做什么（一期）：** 不做第三方登录（微信/Google），不做管理员角色，不做邀请机制。

---

## 四、一期优先级汇总

### P0（必须交付）

- 添加/删除博主（粘贴链接自动识别）
- B 站 + YouTube 视频抓取（每 30 分钟）
- Web 信息流（卡片展示 + 时间线排序 + 点击跳转 + 平台筛选）
- 飞书新视频推送（单条推送 + Webhook 配置 + 去重）
- 用户注册/登录

### P1（优先做，可小幅延后）

- 博主备注名编辑
- 按博主开关通知
- 已读标记
- 收藏功能
- 分页加载
- 抓取降级（推送主页链接）

### P2（有时间再做）

- 飞书免打扰时段

## 五、扩展性预留

一期只做视频源和飞书通知，但设计时需为后续迭代预留扩展空间：

| 扩展方向 | 预留方式 |
|----------|----------|
| 新增信息源类型（资讯、券商数据） | 信息源采用统一的适配器接口，新平台只需实现该接口 |
| 新增视频平台（抖音、小红书等） | 同上，适配器模式 |
| 新增通知渠道（微信、邮件等） | 通知模块抽象化，每个渠道实现统一接口 |
| AI 分析处理 | 内容数据保留原始数据字段，便于后续 LLM 处理 |
| 嵌入式视频播放 | 内容数据中保留平台视频 ID，后续可生成嵌入播放器 |

## 六、核心用户流程

### 6.1 首次使用

用户注册 → 登录 → 配置飞书 Webhook → 添加关注博主（粘贴链接） → 系统立即抓取一次 → 信息流展示视频 → 后续新视频自动推送到飞书

### 6.2 日常使用

打开 FinFlow 网站 → 浏览信息流中的新视频 → 点击感兴趣的视频跳转观看 → 标记已读或收藏

### 6.3 接收通知

博主发布新视频 → 系统 30 分钟内检测到 → 飞书群收到推送消息 → 点击链接直接观看 或 打开 FinFlow 查看更多
